# Diogo Oliveira
### Systems & UX Thinking, Discovered Through Narrative Design

Part of a larger body of work. **[Start here →](https://github.com/VediRago/diogo-oliveira)**

This repo documents systems and UX thinking on its own terms — information architecture, content systems, and progressive disclosure design — independent of the fiction that produced it.

**[Narrative Design Portfolio →](https://github.com/VediRago/diogo-oliveira-portfolio)** — the methodology and case studies this thinking was extracted from.

---

## What's Here

### 1. [Consistency](./case-studies/consistency.md)
The goal governing every navigation decision in this ecosystem: nobody should have to relearn how to find things more than once. Applied directly through **[Layering as a Method →](./case-studies/the-layering-principle.md)**

### 2. [Layering as a Method](./case-studies/the-layering-principle.md)
One method used repeatedly to organize and deploy information, in service of the goal above — flexing between strict build-order and simultaneous-optional depth depending on what the problem needs, shown across five different cases.

### 3. [Blueprint Systems](./case-studies/blueprint-systems.md)
Different artifacts need different information to reach the same goal — a blueprint per artifact type, all answering the same question: what must exist before this is complete.

### 4. [Form as Signal](./case-studies/form-as-signal.md)
The medium's own texture used as a communication channel — prose density thinning to mirror a character's loss of perception, with nothing ever named directly. Not sequencing what a reader sees, but treating *how* something is presented as content in its own right.

### 5. [Designing the Invisible System](./case-studies/invisible-system.md)
A working prototype of a hidden rules engine: a state machine with deliberate instability and a "partial access, never total" data-retrieval model, confirmed firing in both directions with real numbers. Architecture thinking, demonstrated in a built, testable artifact.

### 6. [Scene Blueprint Template](./case-studies/scene-blueprint-system.md)
A documentation system separating content intent from execution, enforcing single-source-of-truth for recurring rules. Schema design and DRY discipline, independently arrived at.

### 7. [People Before System](./case-studies/people-before-system.md)
A sequencing principle: build and present complex systems in the order someone actually discovers them, not the order the architecture was constructed in. Journey-first information architecture.

### 8. [Treasure → Key → Treasure](./case-studies/treasure-key-treasure.md)
A content-layering model where every stage of disclosure must be independently rewarding before it unlocks the next. Progressive disclosure, done more rigorously than most products attempt.

### 9. [Stretching](./case-studies/stretching.md)
A model for updating someone's understanding without invalidating what they already believed. Mental model management for onboarding and features that reveal complexity over time.

### [Process Log](./process-log/)
Day-by-day development entries — the reasoning behind structural decisions, timestamped as they happened.
