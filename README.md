# Diogo Oliveira
### Systems & UX Thinking, Discovered Through Narrative Design

Everything in this repository was developed in roughly 40 days alongside full-time work at a glass factory, while writing an animated series called *Noeme*. No design background. No formal UX training. The thinking below wasn't borrowed from a course — it was built to solve real problems in the writing, then recognized afterward as design discipline.

This repo documents that thinking on its own terms: as information architecture, content systems, and progressive disclosure design — independent of the fiction it came from.

- **[Noeme](https://github.com/VediRago/Noeme)** — the pilot project these systems were built to serve
- **[Narrative Design Portfolio](https://github.com/VediRago/diogo-oliveira-portfolio)** — the full creative practice and case studies

## What's Here

### 1. [Scene Blueprint Template](./case-studies/scene-blueprint-system.md)
A documentation system that separates content intent from execution, enforces single-source-of-truth for recurring rules, and plans for future insertion before it's needed. Includes a side-by-side example showing the system actually producing finished, traceable output.

**This is design documentation, schema design, and DRY discipline — independently arrived at.**

### 2. [People Before System](./case-studies/people-before-system.md)
A sequencing principle: present and build complex systems in the order someone actually discovers them, not the order the underlying architecture was constructed in.

**This is journey-first information architecture.**

### 3. [Treasure → Key → Treasure](./case-studies/treasure-key-treasure.md)
A content-layering model where every stage of disclosure must be independently rewarding before it's allowed to unlock the next stage — rejecting the common pattern of early content existing only as a funnel toward later content.

**This is progressive disclosure, done more rigorously than most products attempt.**

### 4. [Stretching](./case-studies/stretching.md)
A model for updating someone's understanding without invalidating what they already believed. The distinction: *"You were wrong"* vs. *"You were right, but the world is larger than that."*

**This is mental model management — critical for onboarding, feature rollout, and any product that reveals complexity over time.**

### 5. [Designing the Invisible System](./case-studies/invisible-system.md)
The hidden rules engine underneath the visible story — a state machine with deliberate instability, a "partial access, never total" data-retrieval model, and an internal consistency rule that holds even in places no reader will ever question it.

**This is backend/architecture thinking, not just presentation design.**

## Why This Exists As Its Own Repository

The narrative design portfolio shows finished creative work. This repository shows the **process and structural thinking underneath it** — the part that transfers directly to product, content, and UX problems regardless of medium.

Same person, same 40 days, two different audiences.
