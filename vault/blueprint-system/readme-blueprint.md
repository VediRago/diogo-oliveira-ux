# README Blueprint

## Purpose

A README is a repository's entry point. It has to orient a reader before anything else exists for them to click into — what this is, why it exists, where to go next.

## Content

**Thesis or identity line** — stated once, briefly, at the top.

**What's here** — a short framing sentence, then the actual contents as links, ordered deliberately rather than by creation date.

**Status** — what's finished, what's pending, stated plainly.

**Contact or exit** — how to reach the person behind the repository, or where to go next if this isn't what the reader was looking for.

## Rule

A README never repeats content that belongs to a deeper page. It orients; it doesn't explain. If a reader needs the full picture before deciding whether to click a link, the link description has failed, not the reader's patience.
