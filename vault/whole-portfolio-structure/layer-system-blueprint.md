# Layer System Blueprint

## Purpose

The portfolio is organized into layers because different parts of the work answer different questions, and forcing them into one shape would flatten that difference.

This blueprint does not govern what goes inside a layer. It governs why the layers exist, what separates one from another, and when a layer is allowed to break the rules that govern the others.

Every layer-specific blueprint (Layer 2 Blueprint, Case Study Blueprint, future blueprints) sits underneath this one.

## The Core Question

Each layer exists to answer one distinct question for the reader:

- **Layer 1** — Who is this, and where do I go from here?

- **Layer 2** — How is this work approached?

- **Layer 3** — What does that approach produce?

- **Layer 4** — What is this singular thing, on its own terms?

A reader should always know which question a given repository is answering. If a repository tries to answer more than one, it belongs in the wrong layer.

## Layer 1 — Identity

The hub. Introduces the person, not the work.

Contains no methods, no case studies, no products — only orientation and links outward.

Every other layer is reachable from here. Layer 1 is reachable from nowhere else; it is the entry point by definition.

## Layer 2 — Method

One professional domain per repository. Governed by the Layer 2 Blueprint.

Answers "how is this work approached" through Principles and Case Studies, with supporting material as proof.

Layer 2 repositories share a consistency of structure because the disciplines they document — UX, narrative design, production systems — are similar enough in shape that shared rules serve them rather than distort them.

## Layer 3 — Product

The written work itself: narratives, stories, completed pieces.

Layer 3 does not explain the work. It presents it.

Layer 3 repositories share structure for the same reason Layer 2 does: they are similar enough in form (long-form narrative product) that consistency serves clarity instead of forcing a shape onto something that doesn't fit it.

## Layer 4 — Singular Projects

Reserved for work that mixes structure and product in a way no shared blueprint could describe without distorting it.

**Layer 4 is exempt from cross-project consistency rules by design.** Forcing uniformity onto something too singular to generalize would shape the project to fit the system, rather than shaping the system to serve the project. Since the whole architecture exists to remove friction between the work and the reader, applying a rule that fits Layer 2 or Layer 3 to a Layer 4 project would work against the purpose the rule was built to serve.

Each Layer 4 project defines its own internal rules. Those rules do not need to resemble any other project's rules, in Layer 4 or elsewhere.

A project only belongs in Layer 4 if it genuinely cannot be described as either "a method" (Layer 2) or "a product" (Layer 3) without losing what makes it what it is.

## Cross-Layer Rules

These apply everywhere, including Layer 4:

- Every document has one purpose.

- Evidence over explanation. Link to proof rather than restating it.

- Every link points to something real. A placeholder is a broken claim.

- A reader should be able to stop at any layer and have understood something complete, not a fragment waiting on a later layer to make sense.

## Reader Journey

Layer 1 (Who / where to go) ↓ Layer 2 (How the work is approached) ↓ Layer 3 (What that approach produces)

Layer 4 stands apart — reachable from Layer 1 directly, answering its own question on its own terms.

## Completion Checklist

Before considering the layer system itself complete:

- Every existing repository can be placed in exactly one layer without ambiguity.

- Every layer answers a distinct question, with no overlap.

- Layer 2 and Layer 3 each have their own blueprint governing internal consistency.

- Layer 4's exemption is stated explicitly, not left implicit.

- A reader entering at Layer 1 can reach every other layer; no layer is an orphan.

- Every layer can be understood independently while naturally leading to deeper layers.
