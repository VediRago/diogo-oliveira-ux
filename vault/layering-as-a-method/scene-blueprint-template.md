# Scene Blueprint Template

Use this template for scene blueprints, adaptation notes, and script planning.

These files are not prose replacements.

They exist to preserve scene purpose, emotional movement, visual logic, structural function, action flow, dialogue placement, cue interpretation, and future script / animation / show adaptation details.

## File Naming Convention

Use numbered scene files with gaps of 10 so new scenes can be inserted later without renaming everything.

Inside an `01_Scenes` folder, do not repeat `Scene_` in the filename. The folder already defines the file type.

Format:

```text
10_Title_Of_Scene.md
20_Title_Of_Scene.md
30_Title_Of_Scene.md
```

If a new scene is added between two existing scenes, use the gap:

```text
10_Leaving_Church.md
15_Church_Steps_Conversation.md
20_Walk_To_Street_Crossing.md
```

Use the number for order and the title for meaning.

Do not rely on title alone for ordering.

## Act Summary vs Scene Blueprint

Use the act summary for repeated global rules.

Act summaries should hold:
- act purpose
- what happens across the act
- world / design language
- performance language
- recurring motifs
- bridge notes
- scene count and scene map
- open work

Scene blueprints should hold only scene-specific execution.

Do not repeat broad act rules in every scene unless the scene uses them in a specific way.

## Scene Building Method

Build each scene in this order:

1. Purpose
2. World / Design Language, only if scene-specific design is needed
3. Function sections
4. Visual / Camera Function, if needed
5. Action Flow
6. Dialogue / Cue Attachments
7. Key Images / Key Lines / Notes
8. Scene Summary

Put **Scene Summary** at the end so it becomes a clean readable recap after the structure is defined.

## Action Flow vs Dialogue / Cue Attachments

Use **Action Flow** for the clean sequence of what physically happens over the next few seconds.

Use **Dialogue / Cue Attachments** to attach spoken lines, small gestures, acting cues, emotional interpretation, camera attention, and important silent details to the Action Flow numbers.

Only include a `Dialogue:` label when someone speaks. Do not write `Dialogue: None`. Silent beats can contain only `Cue:`.

## Template

```
# Scene — [Scene Title]

Status:
- Scene blueprint
- Prose not written / Prose exists
- Adaptation/script version not written

Act:
- Act [Number] — [Act Name]

Characters:
- [Character 1]
- [Character 2]

Location:
- [Place / room / world / setting]

Purpose:
- [What this scene exists to do emotionally or structurally.]
- [What relationship, pressure, bridge, or motif it builds.]

Emotional Function:
- Start emotion: [What the scene begins with.]
- Shift: [What changes during the scene.]
- End emotion: [What the scene leaves behind.]

Character Function:
- [Character 1]: [What this scene shows or changes about them.]

Structural Function:
- [How this scene connects to the act / bridge / later payoff.]

Visual / Camera Function:
- [Optional. Use when the scene has important visual withholding, framing, reflection, gaze, or shot logic.]

## Action Flow

F01 — [Clean physical/action beat.]
F02 — [Clean physical/action beat.]

## Dialogue / Cue Attachments

F01:
- Cue:
  - [Small gesture, acting note, emotional interpretation, or silent cue.]

F02:
- Dialogue:
  - [Character: "Line."]
- Cue:
  - [How the line attaches to the action.]

Key Images:
- [Image / object / gesture / visual motif.]

Key Lines:
- "[Important line.]"

Important Notes:
- [Rule for writing/adapting the scene.]

Scene Summary:
[Short paragraph explaining what happens in the scene from beginning to end.]

Open Questions:
- [Name to decide later.]
```
