# Worldbuilding Method Hierarchy (Summary)

## Core Rule

> Do not extend the chain unless the next word is produced by everything before it.

Complexity is never added because it would be interesting. It's added only once what already exists has organically demanded it.

## The Four Tiers

1. **Character / background** — `experience → emotion → reason → behavior`
   The base unit. A character doesn't need anything more than this to be valid.

2. **Place / culture** — `... → behavior → atmosphere`
   Earned once behavior repeats around more than one person, becoming the emotional weather of a place.

3. **History / institution** — `... → atmosphere → history`
   Earned once atmosphere survives long enough to harden into law, calendar, tradition, or myth.

4. **Quest / gameplay output** — `... → atmosphere → quest logic`
   Earned once history and atmosphere naturally create needs, obstacles, choices, and consequences — never invented as an objective first.

Not every element needs to reach every tier. A background character may only need tier one. A civilization-defining system usually needs history. A playable quest needs the full chain to reach quest logic naturally, or it reads as an errand pasted onto lore rather than an expression of what already exists.

## Worked Example — Chip Removal Quest

The quest wasn't invented first and justified backward. It exists because four prior layers already demanded it: a founding theft, an obedience-based institution, medicine turned into bodily governance, and chips normalized as identity. The quest is history becoming playable pressure — not a separate formula.

## Design Goal

The goal is immersion: nothing feels pasted on, because every layer follows the same cause-and-effect logic as the one before it. The method isn't about adding more. It's about knowing when the next word is justified.
