# Layering as a Method

*One of the applied methods behind [the repository's principles](../PRINCIPLES.md).*

## The Method

Some information has to exist before other information can be written.

Sometimes that order matters. A later layer cannot work until the earlier layer exists.

Sometimes every layer already exists, and the only question is how deeply the reader chooses to explore.

Layering separates those cases.

---

## Case: Content Depth (Worldbuilding)

Worldbuilding was added only when the story demanded it.

A character did not get atmosphere until behavior repeated. A place did not get history until atmosphere stayed consistent long enough to need one.

→ Method: [Worldbuilding Method Hierarchy](../vault/layering-as-a-method/worldbuilding-method-hierarchy.md)

## Case: Scene Blueprint (Production)

Production documents use a fixed order.

The act defines global rules. The scene blueprint references those rules. The screenplay executes the scene.

Each layer depends on the one before it.

→ Template: [Scene Blueprint Template](../vault/layering-as-a-method/scene-blueprint-template.md)
→ Global: [Act 01 — Lucreta](../vault/layering-as-a-method/act-01-lucreta.md)
→ Local: [Bedroom Preparation](../vault/layering-as-a-method/bedroom-preparation.md)
→ Screenplay: [Lucreta](../vault/layering-as-a-method/lucreta.md)
