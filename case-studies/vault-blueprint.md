# Vault Blueprint

## Purpose

A Vault holds the evidence supporting a case study — real artifacts a reader can go verify — without cluttering the case study itself or the repository's front door.

## Content

**Ownership** — one Vault folder per case study. If more than one case study needs the same underlying artifact, each gets its own copy in its own folder, same filename, no cross-linking.

**Return link** — the first line of every file, before any content, pointing back to the owning case study.

**Artifacts** — evidence only. No explanation duplicated from the case study that already covers it.

## Rule

A Vault is never mentioned in a README. It exists for anyone who goes looking, and does its job correctly precisely by staying invisible to anyone who doesn't.
