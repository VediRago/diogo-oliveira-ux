# Documentation Is UX

Documentation is another interface.

The same principles apply.

---

## One Document. One Job.

README introduces the repository.

Case studies explain methods.

Vaults contain evidence.

Blueprints define structure.

Process logs record development.

---

## Solve One Problem.

Every document answers one question.

If it answers two, split it.

---

## Document Decisions.

Don't document what the reader already learns by using the repository.

Document the decision behind it.

---

## Describe. Don't Sell.

Describe what exists.

Describe why it exists.

Don't explain why it's good.

---

## Fewer Words.

If removing words doesn't change the meaning, remove them.

Less reading.

Less effort.

Less friction.

---

## No Pedigree Prose.

Don't write to sound intelligent.

Don't write to sound important.

Write to communicate.

---

## One Paragraph. One Idea.

One paragraph should answer one question.

---

## Navigation Is Documentation.

The next step should be obvious.

The previous step should be reachable.

Broken links are documentation bugs.

---

## Stop Early.

When the point is made, stop.
